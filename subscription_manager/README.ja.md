# サブスクリプション管理システム

M365（SharePoint リスト + Power Automate）でチームのサブスクリプション契約を一元管理し、請求当日に登録アドレスへ自動通知します。

## できること

- **一覧管理**：サービス名・月額・通貨・課金周期・次回請求日・契約者・カテゴリを SharePoint リストで一元管理
- **請求当日通知**：毎朝 9:00 に本日請求分を抽出し、各サブスクの登録メールアドレスへ通知メールを自動送信
- **次回日付の自動繰り上げ**：通知送信後、課金周期に応じて次回請求日を自動更新（月次→+1ヶ月、年次→+1年、四半期→+3ヶ月）
- **為替換算合計（オプション）**：週1回 USD/EUR レートを更新し、JPY 換算の月額合計を表示
- **チーム共有**：SharePoint の権限機能で閲覧・編集を制御、Teams タブに埋め込み可能

## 前提

- M365 ライセンス（**Power Automate 標準ライセンス**を含むこと）
- 対象チーム全員が M365 アカウントを持っている
- SharePoint サイトへのアクセス（新規作成する場合は作成権限、既存サイトを間借りする場合は編集権限）

## セットアップの流れ

| Phase | 内容 | 所要 | 必須/任意 |
|------|------|-----|---------|
| 1 | SharePoint サイトの選定・準備 | 5〜15分 | 必須 |
| 2 | Subscriptions リスト作成と列定義 | 25分 | 必須 |
| 3 | ExchangeRates リスト作成 | 5分 | オプション |
| 4 | 通知フロー作成（Flow 1） | 30分 | 必須 |
| 5 | 為替更新フロー作成（Flow 2） | 15分 | オプション |
| 6 | テスト実行 | 10分 | 必須 |
| 7 | チームメンバー招待 | 10分 | 必須 |

**最小構成（Phase 1, 2, 4, 6, 7）で 約1時間 20分**。JPY のサブスクしか無い or「月合計 JPY 換算」が要らない場合は Phase 3・5 を丸ごとスキップ可能。

## Phase 1: SharePoint サイトの選定・準備

**A（推奨）と B のどちらかを選択**します。所内展開のスピード重視なら **A（既存サイト間借り）** が圧倒的に楽です。

### A（推奨）: 既存のチームサイトを間借りする

多くの M365 テナントで、**サイトの新規作成は管理者権限**が必要です。既にあるチームサイト（Teams チャネル背後のサイト、部門サイトなど）にリストを追加するだけなら、通常の編集権限で完結します。

1. 使用する既存サイトを決める（例：所属チームの Teams チームの SharePoint サイト）
2. サイトの URL を控える：`https://<テナント>.sharepoint.com/sites/<既存サイト名>/`
   - 以降、この URL を `{SITE_URL}` と表記
3. **注意**：この既存サイトに **アクセスできる人全員**にリストが見えます。サブスク情報を絞りたい場合は、Phase 7 の「リスト単位の権限分離」を参照

### B: 新規サイトを作成する（要作成権限）

サブスク管理を専用サイトで運用したい／既存サイトの閲覧者が広すぎる場合。

1. https://www.office.com にサインイン
2. アプリランチャー（左上の 9つの点）→ **SharePoint** を選択
3. 左上の **「＋ サイトの作成」** をクリック（**このボタンが出ない場合は作成権限が無い**ので A に切り替え）
4. **「チーム サイト」** を選択
5. サイト名: `サブスクリプション管理`（任意）
6. グループのエイリアス: `subscription-mgmt`（メールアドレスに使われる、半角英数）
7. 言語: **日本語** / タイムゾーン: **(UTC+9:00) 大阪、札幌、東京**
8. プライバシー設定: **プライベート**（メンバーのみアクセス可）
9. **「次へ」** → 追加のメンバーは Phase 7 で追加 → **「完了」**
10. 作成後の URL を控えて `{SITE_URL}` として使用

## Phase 2: Subscriptions リスト作成

### まず最重要：「内部名」と「表示名」の話

SharePoint の各列には **内部名** と **表示名** の2種類の名前があります。

|  | 表示名 | 内部名 |
|---|---|---|
| 用途 | 画面に表示される名前 | Power Automate のフィルタ・URL・API で使う識別子 |
| 変更 | いつでも可能 | **列作成時に決まったら二度と変更不可** |
| 例 | `月額` | `Amount` |

もし列を作るときに直接 `月額` と入力すると、**内部名は `_x6708__x984d_` のような日本語エンコード文字列**になります。Power Automate では動くものの、フィルタや式が読めなくなります。

**回避策**：**列を英語名で作ってから、表示名だけ日本語にリネーム**。すると内部名は英語のまま、画面表示は日本語になります。

#### 具体的手順（各列に対して2ステップ）

**ステップ 1: 英語名で列を作る**
1. **「＋ 列の追加」** をクリック
2. 型を選ぶ（例：通貨、選択肢）
3. 右パネルの **「名前」** に **英語で** 入力（例：`Amount`）
4. **「保存」**

**ステップ 2: 表示名を日本語にリネーム**
1. ヘッダーに表示された英語名（例：`Amount`）をクリック
2. **「列の設定」** → **「名前の変更」**
3. 日本語名（例：`月額`）を入力して保存

これで **表示は「月額」、内部名は `Amount`** の状態になります。

> **既に日本語名で作ってしまった列は？** データが入っていなければ **削除して英語名で作り直す**のが最も速いです（各列の「列の設定」→「この列を削除」）。既にデータが入っている場合は、**⚙ 設定 → リストの設定 → 列名クリック → URL の `Field=...` の値**が内部名なので、それを使って Power Automate 側を書けば動きます。

> **`Title` 列だけは例外**：デフォルトで存在するタイトル列（`サービス名` として使う列）は、表示名を何に変えても内部名は必ず `Title` です。ここだけは日本語で作っても大丈夫。

### リスト作成

1. **`{SITE_URL}`** の左メニューから **「サイト コンテンツ」** をクリック
2. **「＋ 新規」** → **「リスト」** → **「空のリスト」**
3. 名前: `Subscriptions`（**必ず英語名で**。フロー内での参照が安定します）
4. 説明: `サブスクリプション契約の一元管理リスト`
5. 「サイトのナビゲーションに表示」: **ON**
6. **「作成」** をクリック

### 列定義

まず **既定の `Title` 列** を「サービス名」にリネーム：
1. `Title` 列のヘッダー → **「列の設定」** → **「名前の変更」** → `サービス名`

続けて、以下の11列を追加します。**上のステップ1（英語で作成）→ ステップ2（日本語にリネーム）** を各列で繰り返します。

| # | 表示名（後で変更） | 内部名（作成時にこれを入力） | 型 | 詳細設定 | 必須 |
|---|-------|-------|-----|---------|------|
| 1 | サービス名 | Title | 1行テキスト | (Title を流用) | ✓ |
| 2 | 月額 | `Amount` | 通貨 | 小数点第2位 / 既定=0 | ✓ |
| 3 | 通貨 | `Currency` | 選択肢 | JPY, USD, EUR / 既定=JPY / 単一選択 | ✓ |
| 4 | 課金周期 | `BillingCycle` | 選択肢 | 月次, 年次, 四半期 / 既定=月次 / 単一選択 | ✓ |
| 5 | 次回請求日 | `NextBillingDate` | 日付と時刻 | **日付のみ** に設定 | ✓ |
| 6 | 通知先メール | `NotifyEmail` | 1行テキスト | メールアドレス | ✓ |
| 7 | 通知先CC | `NotifyCc` | 1行テキスト | 複数は `;` 区切り | |
| 8 | カテゴリ | `Category` | 選択肢 | AI, 動画配信, 業務ソフト, クラウド, その他 | |
| 9 | 契約者 | `Owner` | ユーザーまたはグループ | ユーザーのみ / 単一選択 | ✓ |
| 10 | メモ | `Notes` | 複数行テキスト | プレーンテキスト / 6行 | |
| 11 | 解約URL | `CancelUrl` | ハイパーリンクまたは画像 | URL | |
| 12 | 有効 | `Active` | はい/いいえ | 既定=はい | ✓ |

**効率的な進め方**：11列すべて英語名で作り切ってから、まとめて日本語表示名にリネームすると早いです。

### ビュー作成（見え方の切り替え）

上部の **「すべてのアイテム ▼」** → **「現在のビューの変更」** → 保存し直しで作れます。

- **今月請求**: フィルタ `NextBillingDate <= [今日+30日] AND Active = はい`、並び順 `NextBillingDate 昇順`
- **自分の分**: フィルタ `Owner = [自分] AND Active = はい`
- **無効化済み**: フィルタ `Active = いいえ`

## Phase 3: ExchangeRates リスト作成（オプション）

> **必要な場合のみ**：「月額合計を JPY 換算して見たい」場合のみ作成します。基本の請求日通知（Flow 1）には**一切関係ありません**。JPY オンリー運用ならスキップ、まずは Phase 4 に進んでください。

為替レートを保持する小さなリスト。Phase 5 の Flow 2 が週1回上書きします。

1. 同じサイトで **「＋ 新規」** → **「リスト」** → **「空のリスト」**
2. 名前: `ExchangeRates`
3. 「サイトのナビゲーションに表示」: **OFF**（内部利用）
4. `Title` を **通貨コード** にリネーム
5. 列追加（内部名を英語で作ってから表示名を日本語に変更）：

| 表示名 | 内部名 | 型 | 詳細設定 |
|-------|-------|-----|---------|
| 通貨コード | Title | 1行テキスト | USD / EUR など |
| JPYレート | `Rate` | 数値 | 小数点第4位 |
| 最終更新 | `LastUpdated` | 日付と時刻 | 日付と時刻 |

6. 初期データを手入力：
   - `USD` / `157.0` / (今日の日付)
   - `EUR` / `168.0` / (今日の日付)

## Phase 4: 通知フロー作成（Flow 1: Daily Billing Notification）

### フロー作成開始

1. https://make.powerautomate.com にアクセス
2. 左メニュー **「マイ フロー」** → **「＋ 新しいフロー」** → **「スケジュール済みクラウド フロー」**
3. フロー名: `Subscription Daily Notification`
4. 開始日時: 明日の **9:00** / 繰り返し: **1日おき**
5. **「作成」**

### アクション構築

#### アクション 1: リストから本日請求分を取得

- **「＋ 新しいステップ」** → 検索: `SharePoint` → **「複数の項目の取得」**
- サイトのアドレス: `{SITE_URL}` を選択
- リスト名: `Subscriptions`
- **「詳細オプションを表示」** をクリック
- **Filter Query**:
  ```
  NextBillingDate eq '@{formatDateTime(convertTimeZone(utcNow(), 'UTC', 'Tokyo Standard Time'), 'yyyy-MM-dd')}' and Active eq 1
  ```

#### アクション 2: 各行を処理

- **「＋ 新しいステップ」** → **「Apply to each」**（コントロール）
- 「以前の手順から出力を選択」: 前ステップの **value** を選択

以下、Apply to each の**内側**にアクションを追加：

##### 2-1: メール送信

- **「＋ アクションの追加」** → 検索: `Outlook` → **「メールの送信 (V2)」**
- 宛先: 動的コンテンツから **NotifyEmail**
- **「詳細オプションを表示」**
- CC: 動的コンテンツから **NotifyCc**
- 件名:
  ```
  【本日請求】@{items('Apply_to_each')?['Title']} @{items('Apply_to_each')?['Amount']} @{items('Apply_to_each')?['Currency']?['Value']}
  ```
- 本文（HTML モードに切り替えて `templates/email_notification.html` の内容を貼り付け、動的コンテンツを差し込み）

##### 2-2: 次回請求日を更新

- **「＋ アクションの追加」** → 検索: `SharePoint` → **「項目の更新」**
- サイトのアドレス: `{SITE_URL}`
- リスト名: `Subscriptions`
- ID: 動的コンテンツから **ID**
- サービス名: 動的コンテンツから **Title**
- 通貨: 動的コンテンツから **Currency Value**
- 課金周期: 動的コンテンツから **BillingCycle Value**
- 契約者: 動的コンテンツから **Owner Claims**
- 次回請求日:
  ```
  @{if(equals(items('Apply_to_each')?['BillingCycle']?['Value'], '月次'),
        addToTime(items('Apply_to_each')?['NextBillingDate'], 1, 'Month'),
     if(equals(items('Apply_to_each')?['BillingCycle']?['Value'], '年次'),
        addToTime(items('Apply_to_each')?['NextBillingDate'], 1, 'Year'),
        addToTime(items('Apply_to_each')?['NextBillingDate'], 3, 'Month')))}
  ```
- Amount, NotifyEmail, Active など他の必須項目も動的コンテンツから元の値をそのまま渡す

> **月末の丸め**: `addToTime('2026-01-31', 1, 'Month')` は `2026-02-28` を返します。1月末の契約は 2月28日 → 3月28日 と繰り上がる点に注意。

### 保存とテスト

1. 右上 **「保存」**
2. **「テスト」** → **「手動」** → **「保存とテスト」**
3. 実行結果を確認：各ステップが緑チェックなら OK
4. リスト上で次回請求日が更新されていることを確認

## Phase 5: 為替更新フロー作成（Flow 2: Weekly FX Update）（オプション）

> **Phase 3 を作成した場合のみ**。JPY オンリー運用ならスキップ。

### フロー作成

1. **「＋ 新しいフロー」** → **「スケジュール済みクラウド フロー」**
2. フロー名: `Subscription FX Update`
3. 開始日時: 次の月曜 **8:00** / 繰り返し: **1週間おき**
4. **「作成」**

### アクション構築

#### アクション 1: 為替レートを取得（USD）

- **「＋ 新しいステップ」** → 検索: `HTTP` → **「HTTP」**（Premium コネクタ）
  - **Premium ライセンスがない場合**：HTTP アクションが使えないので、代替として為替レートを手動更新する運用にする（ExchangeRates リストを月1回目視で更新）。以降のアクションはスキップ。
- 方法: `GET`
- URI: `https://api.exchangerate.host/latest?base=USD&symbols=JPY`

#### アクション 2: JSON 解析

- **「JSON の解析」**（データ操作コネクタ）
- コンテンツ: 前ステップの **本文**
- スキーマ:
  ```json
  {
    "type": "object",
    "properties": {
      "rates": {
        "type": "object",
        "properties": {
          "JPY": { "type": "number" }
        }
      }
    }
  }
  ```

#### アクション 3: ExchangeRates リストの USD 行を更新

- **「項目の更新」** (SharePoint)
- サイトのアドレス: `{SITE_URL}`
- リスト名: `ExchangeRates`
- ID: USD 行の ID（**取得アクション**を先に噛ませて動的に取得しても良いし、リスト作成時の ID を固定値で入れても良い）
- 通貨コード: `USD`
- JPYレート: JSON 解析結果の **JPY**
- 最終更新: `@{utcNow()}`

#### アクション 4-6: EUR も同様に処理

（HTTP → JSON 解析 → 項目の更新 の3ステップを EUR 用にコピー）

### 保存とテスト

1. **「保存」** → **「テスト」** → **「手動」** で実行
2. ExchangeRates リストのレートが更新されていることを確認

## Phase 6: テスト（エンドツーエンド）

1. Subscriptions リストに **テスト行**を1つ追加
   - サービス名: `テスト通知`
   - 月額: `1000`
   - 通貨: `JPY`
   - 課金周期: `月次`
   - 次回請求日: **今日の日付**
   - 通知先メール: **自分のアドレス**
   - 契約者: **自分**
   - 有効: **はい**
2. **Flow 1（Subscription Daily Notification）を手動実行**
3. 期待結果：
   - 自分のメール受信箱に「【本日請求】テスト通知 1000 JPY」が届く
   - リストの `テスト通知` 行の次回請求日が**来月の同日**に更新されている
4. テスト行は削除するか、次回請求日を1年後にして無効化しておく

## Phase 7: チームメンバー招待

### 選択肢 A（Phase 1 で新規サイトを作成した場合）

サイト全体へのアクセス権を付与：

1. サイト右上 **⚙ 設定** → **「サイトの権限」**
2. **「メンバーの追加」** で M365 アカウントを追加
3. 権限レベル：
   - **編集**：新規サブスク追加・自分の契約情報を編集
   - **表示**：閲覧のみ（一般メンバー向け）
   - **フル コントロール**：自分のみ

### 選択肢 B（既存サイトを間借りしていて、閲覧者を絞りたい場合）

**リスト単位で権限を切る**方法：

1. `Subscriptions` リストを開く → 右上 **⚙ 設定** → **「リストの設定」**
2. **「このリストの権限」** をクリック
3. 上部リボンの **「アクセス許可の継承を中止」** → 確認ダイアログで **OK**
4. これでリスト独自の権限になるので、不要なメンバーを削除・必要なメンバーだけ追加
5. `ExchangeRates` リストにも同じ手順を適用

## 運用のポイント

### 新規サブスク追加

1. SharePoint リスト画面右上 **「＋ 新規」**
2. 各項目を入力（必須項目に注意）
3. **「保存」**

### 解約時

- `有効` を **いいえ** に切り替える（履歴として残す）
- 完全に削除したい場合は行を削除

### 為替換算の月額合計を確認したい（Phase 3・5 を作成済みの場合）

SharePoint に**計算列**を追加：
- 列名: `月額JPY`
- 数式: `=IF([通貨]="JPY", [月額], [月額]*<USDレート>)`
- ※ 計算列は他リストを参照できないので、USDレートは手動更新するか、Power BI / Excel Online で外部集計する必要あり
- 簡単な代替：Excel Online で Subscriptions リストをテーブルとして開き、ピボットや `SUMPRODUCT` で集計

### トラブルシュート

- **メールが届かない** → Flow 1 の実行履歴で失敗ステップを確認。多くは「NotifyEmail 列が空」か「Outlook の送信権限エラー」
- **Filter Query が構文エラー** → 内部名が日本語エンコード（`_x6708__x984d_` など）になっている可能性。`⚙ 設定 → リストの設定 → 列名クリック` で URL の `Field=` を確認
- **次回請求日の月末繰り上げが期待と違う** → `addToTime` の月末丸め仕様（末日→末日）を確認
- **HTTP アクションが使えない（Premium 要求）** → Flow 2 は諦めて ExchangeRates を月1手動更新にする
- **`+ サイトの作成` ボタンが SharePoint トップに出ない** → 作成権限がないので Phase 1 A（既存サイト間借り）に切り替え

## ライセンス / 情報
